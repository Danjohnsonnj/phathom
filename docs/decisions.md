# Phathom — Decision Log

Running record of architectural and product decisions. Every phase appends here. When starting a new phase, read this entire file before beginning work.

| Date | Decision | Rationale | Phase |
|------|----------|-----------|-------|
| 2026-05-01 | App name is **Phathom** | Mockups used "LinkSavr" as a placeholder; repo name is canonical | Pre |
| 2026-05-01 | **Local-only forever** — no CloudKit, no sync | Simplifies schema (no conflict resolution, no blob size limits), aligns with privacy-first brief | Pre |
| 2026-05-01 | AI engine **TBD** — evaluate Apple FoundationModels vs Llama.cpp in Phase 2 | Both have trade-offs (bundle size vs device requirement); need on-device benchmarks before committing | Pre |
| 2026-05-01 | Delivery is **3 phases**: UI shell → pipeline → RAG chat | De-risks by proving UI/data model before investing in AI integration | Pre |
| 2026-05-01 | **Remove FAB** from library screen; Add New tab is the sole capture entry point | Mockup showed both FAB and tab, which is redundant; one path reduces confusion | Pre |
| 2026-05-01 | `summaryBullets` and `extracts` stored as **JSON strings** on ContentItem, not as separate model relationships | SwiftData relationships for variable-length structured data add complexity with no query benefit; Codable helpers keep it clean | Pre |
| 2026-05-01 | "Key Figures" generalized to **Extracts** — label/value pairs applicable to any content type | Mockup's "Extracted Key Figures" is too narrow; recipes have ingredients, articles have names/dates, photos have descriptions | Pre |
| 2026-05-01 | `processingStatus` uses **granular sub-states** (pending/scraping/embedding/summarizing/tagging/completed/failed) | Enables micro-state labels in the library UI per the "Status Transparency" design principle | Pre |
| 2026-05-01 | **Spotlight + AppIntent** integration added to Phase 2 scope | High-value, low-effort; makes the "personal brain" searchable system-wide | Pre |
| 2026-05-01 | Thumbnail fallback uses **deterministic color from UUID** + content-kind icon | Ensures consistent visual rhythm in library rows even when no image exists | Pre |
| 2026-05-01 | **Llama.cpp is the sole AI engine** — no Apple FoundationModels | Direct experience: FM crashed after a few chat exchanges (4096 token context limit, uncaught error). FM is still beta-unstable (EXC_BAD_ACCESS, dyld errors, guardrail false positives, `@Generable` refusal regressions). FM's ~3B model is weaker than available 8B GGUF options. FM requires iPhone 15 Pro+ so a fallback would be needed anyway. Llama.cpp has been crash-free under extended use. | Pre |
| 2026-05-01 | **User-provided models** — no bundled GGUF files | User obtains `.gguf` files from public model sources, stores them in a location they control (e.g. **On My iPhone** in Files), and selects the file in Settings. **No models ship in the app bundle** (~4.5 GB avoided). Selection persistence and in-place reads use a **security-scoped bookmark** — see **2026-05-02** (not a copy into Phathom’s sandbox Documents). | Pre |
| 2026-05-01 | **Llama-3 chat template only** for v1 prompt format | Support one template well rather than many poorly. Non-Llama-3 models may still work but prompt format is not guaranteed. Document as a future enhancement. | Pre |
| 2026-05-01 | **Graceful JSON parse failure** — partial data, not `.failed` status | If LLM output can't be parsed as JSON for summaryBullets/tags/extracts, set the field to nil but still mark item `.completed`. Better to have an item with a title and no summary than one stuck in `.failed`. | Pre |
| 2026-05-01 | **BGProcessingTask `requiresExternalPower` = false** for analyze | Phase 2 hand-off suggested `true`; **false** lets analysis progress on battery during development and typical mobile use. Revisit if iOS jetsam or thermal issues appear in production. | 2 |
| 2026-05-01 | **No embedding persistence in Phase 2** | `embedding` is a pipeline state only; vector storage requires escalation (new model or external store). RAG/chunking deferred to Phase 3. | 2 |
| 2026-05-01 | **Notes store Markdown in `rawText`** | Keeps schema unchanged while allowing rich note authoring and future markdown-aware rendering. | 2 |
| 2026-05-01 | **Use `swift-markdown-ui` (`MarkdownUI`) for note rendering** | Mirrors Intrai's proven rendering dependency and provides better markdown fidelity than ad-hoc parsing. Approved exception to the no-third-party-package default. | 2 |
| 2026-05-01 | **Archive = soft delete (Option B)** — item vanishes from the library as if deleted; data kept **48 hours** from archive time | Users get Apple-style **Undo** immediately after archiving (toast/sackbar, ~5s) and a **Recently Deleted** screen under **Settings** to restore or delete forever before expiry. Requires `archivedAt: Date?` on `ContentItem`. **Purge**: permanently delete records where `isArchived && archivedAt + 2 days < now` — run on app foreground/active (Phase 1) and in **BGAppRefreshTask** (Phase 2) so items expire even if the app is not opened. **Spotlight** (Phase 2): remove index when archived; re-index on restore if `.completed`. **RAG** (Phase 3): exclude archived items (existing `!isArchived`). | 1–2 |
| 2026-05-02 | **GGUF selection via security-scoped bookmark** — no copy into Phathom Documents | **Product**: one large `.gguf` (~4.5 GB) can live once under **On My iPhone** and be referenced from multiple apps instead of duplicating per-app. **Technical**: `ModelManager` stores `phathom.selectedGGUFBookmark` (`Data` from `URL.bookmarkData`); **`openSelection()`** resolves + **`startAccessingSecurityScopedResource()`**; **`SharedLlamaInference.withSession`** (FIFO **`AsyncLock`**) and **`BackgroundPipeline`** open scoped access for a full inference pass and unload. **Legacy** `phathom.selectedGGUFPath` migrates once. **Constraint**: file must stay **locally materialized**; iCloud‑only / evicted blobs may fail or stall **`BGProcessingTask`** — Settings discloses this. | 2 |
| 2026-05-02 | **Recently Deleted list performance** | **Settings → Recently Deleted** uses **`ScrollView` + `LazyVStack`** (not `List` + card-in-cell), a **`fetchLimit`** on the archived query, and **`ContentCardRow` chrome `.plain`** so rows are not double-wrapped. Reduces main-thread layout churn and large synchronous work when opening the screen. See [`docs/debugging/recently-deleted-freeze.md`](debugging/recently-deleted-freeze.md) (resolved). | 2 |
| 2026-05-02 | **Granular status labels (Library + Detail)** | **`ProcessingStatusPresentation`** maps each `ProcessingStatus` to user-facing copy (Queued, Fetching source, Preparing analysis, Generating summary, Creating tags, Needs attention); hidden when completed. **`DetailView`** and **`ProcessingStatusBadge`** (`ContentCardRow`) share this helper so labels stay consistent and update live with SwiftData. | 2 |
| 2026-05-02 | **Tag/extract prompt budgets + pipeline timing logs** | **Tags**: first **~4k** chars of article, **maxTokens 96**. **Extracts**: first **~8k** chars (**Summary** still **~12k**). **`PipelineMetrics`** prints `scrape`, `load_model`, `summarize`, `tags_llm`, `tag_db`, `extracts_llm` durations to the console (`[PhathomPipeline]` prefix) for profiling; optional removal by deleting the helper and call sites. **Tag DB**: batched fetch/upsert for Llama + hashtag merge tags. | 2 |
| 2026-05-02 | **Tag format & normalization** | Stored tag names are **lowercase ASCII kebab-case** (`a-z`, `0-9`, `-`); multi-word subjects use hyphens. The Llama tag prompt encodes this contract (JSON array, subject + content-type mix); **closed vocabulary for content-type is deferred**. **`TagNameNormalizer`** (PhathomCore) normalizes both LLM output and platform hashtags before upsert. | 2 |
| 2026-05-02 | **Offline-tolerant web ingest** | Saves web URLs locally with **no network** at capture time. Transient **`URLError`** codes on the scrape fetch map to **`WebIngestError.offline`**; items stay **`pending`** with detail **Waiting for network…** (not **failed**). **`NetworkReachability`** (`NWPathMonitor`) triggers **`scheduleForegroundDrain`** / **`scheduleIngest`** when connectivity returns. **`HTTPURLResponse.url`** after redirects drives **`displayHost`** and Instagram/TikTok vs generic routing in **`WebIngestService`**. | 2 |
| 2026-05-02 | **Serialized GGUF session (`AsyncLock` + `withSession`)** | Prevents **`unload()`** / Settings test from racing **`generate*`** between actor `await`s (intermittent **`No model is loaded.`**). Pipeline load→summarize→tags→extracts runs in one locked session; BG task expiration sets **`cancelFlag`** + **`signalCancelInFlight()`** only (no competing unload). | 2 |
| 2026-05-02 | **`sourceMarkdown` on `ContentItem` + SwiftSoup for generic web scrape** | **Display**: structured Source Content in Detail (MarkdownUI) from **generic** HTML without changing **`rawText`** (still flattened plain text for **Llama**, **library search**, **Spotlight**). **`sourceMarkdown`** is optional (~50 KB cap); **Instagram/TikTok** stay markdown-nil. **SwiftSoup** approved for HTML parse (exception alongside **`swift-markdown-ui`**). **No backfill**: existing rows keep **`rawText`**-only Source until re-ingested. | 2 |
| 2026-05-02 | **Readability-style `MainContentExtractor` for generic web** | Both **`rawText`** (LLM/search/Spotlight) and **`sourceMarkdown`** (Detail) now reflect the **same main-content subtree**, picked by a Mozilla Readability-style scorer (text length, comma count, link density, class/id token cues, sibling sweep, fallback to `article`/`main`/`body`). Strips `nav`/`footer`/`aside`/`script`/`style`/`form`/`iframe`/role-banner/role-complementary noise once on the parsed `Document`, then walks the chosen `Element` for both outputs. **Generic web only**; Instagram/TikTok structured payloads untouched. **No backfill**: existing rows keep their original `rawText`/`sourceMarkdown` until re-ingested. **Fallback** to `extractReadableText(from:)` when no candidate clears the `minTextLength = 250` floor, so previously-scrapeable pages don't regress to empty. | 2 |
| 2026-05-03 | **User-editable title via `titleUserSet` flag** | **`ContentItem.titleUserSet: Bool = false`** marks a title as manually entered; the scrape pipeline (**`BackgroundPipeline`**) **never overwrites** `title` when the flag is set. Replaces the brittle host-vs-title heuristic so a manual title equal to the host is also preserved. **Detail** renders an inline **`TextField`** over the title (commits on `.onSubmit` and on focus loss): non-empty trim sets the flag and re-indexes Spotlight; clearing the field resets the flag so the next scrape can repopulate it. **`AddNewTab`** sets the flag for `.web`/`.note` when the user types a title; **`ShareCapture.insertMediaItem`** sets it when callers pass a title. **Lightweight migration** (defaulted `Bool`); **no backfill** for existing rows. | 2 |
| 2026-05-03 | **KV cache prefix reuse + runtime tuning for article analyze** | **Prompts**: article-first (`<ARTICLE>…</ARTICLE>` then task suffix) so all three tasks share an identical token prefix. **Runtime**: decode prefix once on seq 0; per task **`llama_memory_seq_cp`** fork → suffix decode → generate → **`llama_memory_seq_rm`**. **`LlamaContentAnalyzer.analyzeArticle`** + **`ModelSession.analyze`**; **`BackgroundPipeline`** checkpoints via **`onPartial`**. Fallback: **`contextLimitReached`** or stub → sequential **`generate*`** (adaptive fitting). **Context params**: **`n_seq_max = 4`**, **`flash_attn_type = AUTO`**, **`offload_kqv = true`**, **`kv_unified = true`**, **`n_ubatch`** from **`RuntimeConfig.physicalBatchSize`** (default **1024**). **Measured (informal)**: same article, **iPhone 16 Pro**, cold start after force-quit, **~60s → ~30s** wall time for load + full analyze; varies by model/article/thermals. Console: **`[PhathomPipeline]`** stages. | 2 |
| 2026-05-04 | **Foreground = Metal, background = CPU; no opportunistic LLM lane** | iPhone has no Background GPU Access today (`BGTaskScheduler.supportedResources` excludes `.gpu`); Metal work submitted from a `BGProcessingTask` returns `kIOGPUCommandBufferCallbackErrorBackgroundExecutionNotPermitted`, which **before llama.cpp PR #20306** (merged 2026-03-09) triggered `GGML_ABORT` and killed the app. The vendored `llama.xcframework` (built 2026-04-26) includes that fix, but the failure mode is still unpleasant. **Decision**: drop the opportunistic `com.phathom.analyze` `BGProcessingTask` lane entirely; LLM analysis runs on **Metal** (`n_gpu_layers = -1`) only when the app is foregrounded, and on **CPU** (`n_gpu_layers = 0`, `n_ubatch = 256`) inside a user-initiated `BGContinuedProcessingTask` (`com.phathom.continued-analyze`). `BGAppRefreshTask` (`com.phathom.ingest`, no LLM) stays. **Lifecycle hardening**: on `scenePhase != .active` we signal the cancel flag and call `SharedLlamaInference.forceUnloadIfIdle()` so a stale Metal context is never held across a background transition. **`IdleTimerCoordinator`** keeps the screen alive while the foreground drain is active. **`UIBackgroundModes`** reduces to `fetch`. | 2 |
| 2026-05-04 | **Snapshot semantics for "Continue in background"** | Each tap of the LibraryTab "Continue in background" banner freezes the IDs that are currently in `pending` / `embedding` (excluding archived and media) into **`PendingSnapshotStore`** (`UserDefaults` in the app group). The `BGContinuedProcessingTask` handler reads that frozen list and calls **`PipelineWorkGate.processSnapshotItem(id:backend:.cpu)`** for each ID under the same FIFO `AsyncLock` the foreground drain uses. **Items added after the tap stay in their original status** until the next foreground drain or another tap — matches Apple's "tied to an explicit user action" guidance and avoids unbounded background runs. The handler clears the snapshot on completion (or cancellation); the LibraryTab re-reads on `scenePhase == .active` and falls back to the regular submit banner if processable items remain. Cancellation tunnels through `task.expirationHandler` → `cancel.value = true` → `SharedLlamaInference.signalCancelInFlight()` → `llama_set_abort_callback`, exactly the existing cooperative-cancel chain. | 2 |
